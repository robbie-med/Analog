Analog Operating System (AOS)

This manual defines a stable, low-tech life infrastructure: capture → action → record → archive → reference. Digital stays as a servant (communication + storage). Paper becomes the source of truth for cognition and decisions.

What this is

A set of physical containers + rules that keep your life organized for years without apps.

What this is not

Not a productivity hack. Not a journal aesthetic. Not a new dashboard.

Success condition

You can answer the five questions in ≤10 seconds: today, own, done, proof, know.

Quickstart (60 minutes)

  1. Choose containers: 1 command book, 1 record binder, 1 archive box, 1 wall calendar.
  2. Label dividers: Medical, Financial, Insurance, Employment, Legal, Household, Vehicles.
  3. Build the archive index: first sheet inside archive box lists what’s where.
  4. Set your cadence: weekly maintenance (15–25 min) + monthly review (30–45 min).
  5. Enforce the boundary: paper for decisions; digital only for transmission + cold storage.

Golden boundary rule

If it changes your behavior → paper first. If it only informs you → digital permitted.

System architecture

The AOS has five physical layers, each with a time horizon and a strict purpose. Mixing layers causes chaos.

Layer Name Time horizon Purpose Container
1 Capture minutes–days collect inputs pocket notebook / index cards
2 Action days–months run active life command book / action binder
3 Record months–years structured history record binder
4 Archive years–decades original proof archive box (ideally fire-resistant)
5 Reference timeless knowledge lookup reference binder(s)

The five questions

  • What am I doing today? → Command book / Action layer
  • What do I own? → Inventory binder
  • What have I done? → Record binder
  • Where is proof? → Archive box
  • What do I know? → Reference binder

Physical setup

Minimum viable kit

  • Command book (your green book is perfect)
  • Record binder + dividers
  • Archive box + folders
  • Wall calendar (monthly)
  • Pen + pencil + highlighter

Recommended upgrades

  • Fire-resistant archive storage
  • Portable emergency folder
  • Label maker or consistent handwriting
  • Hole punch + staples + paper clips
  • Document sleeves for originals you must handle

One rule prevents 80% of failure

Touch paper once. When you receive it: trash, record, archive, or act—immediately.

Layer 1 — Capture

Purpose: prevent mental RAM overload. Capture is temporary by design.

Capture rules
  • Everything enters the system through capture.
  • Nothing lives here permanently.
  • Process capture daily: transfer, schedule, file, or discard.
  • If it’s not processed within 48 hours, it becomes clutter.
Capture formats

Pocket notebook

  • Best for continuous notes
  • Use short bullets
  • Mark with symbols: ☐ task, ◇ idea, ! urgent, → file

Index cards

  • Best for one-thought-per-card
  • Easy to sort into projects
  • Great for “inbox” processing

Layer 4 — Archive box (proof)

Purpose: retain originals and irreplaceable proof. Archive is sacred: no scratch paper, no mixed notes, no “maybe important later.”

Folder schema (recommended)
  • IDENTITY (birth cert, passport copies, SSN docs)
  • LEGAL (contracts, wills, legal notices)
  • MEDICAL (summaries, key tests, immunizations)
  • FINANCIAL (bank letters, major receipts)
  • INSURANCE (policies, claim summaries)
  • PROPERTY (deeds, leases, titles)
  • TAXES (one folder per year)
  • EMPLOYMENT (offers, evaluations, credentials)
  • CERTIFICATES (licenses, training)
  • WARRANTIES (big-ticket items)

Archive filing rules
  • Originals only (or clearly labeled copies).
  • Sort newest → front.
  • Every archived document should be referenced from the Record binder when relevant.
  • If you remove a document, leave a placeholder note.
Retention guidance (simple)
  • Keep forever: identity, titles/deeds, licenses/certificates, major contracts, tax returns (and key attachments), insurance policies + claim closures.
  • Keep 7 years: most financial proofs, receipts related to taxes, major medical bills.
  • Keep 1–2 years: routine receipts, minor statements, nonessential notices.

If a paper can change your legal/financial status later, keep it longer.

Audit cadence
  • Pantry: monthly quick check
  • Tools/gear: quarterly
  • Big inventory (everything): yearly

Maintenance cadence

Weekly (15–25 minutes)

  • Process capture → action/record/archive/trash
  • Update financial weekly ledger
  • Review calendar + top projects
  • File loose papers

Monthly (30–45 minutes)

  • Reconcile month finances (big categories)
  • Review goals + project outcomes
  • Update maintenance schedules
  • Archive any “proof” documents

Yearly (1–2 hours)

  • Audit archive box + update master index
  • Archive filled notebooks/binders (label spine with date range)
  • Refresh emergency folder
  • Inventory audit

Digital boundary (your vault + files)

Keep digital as cold storage and transmission. Your Obsidian vault becomes a library, not a workspace. Photos can remain digital-native (Ente is fine).

Operating rules
  • Paper is the source of truth for plans, tasks, decisions, logs.
  • Digital is allowed for: messaging, maps, work systems, cold storage, backups.
  • Schedule one “library window” per week to scan/backup/file digitally.
  • No personal tracking apps. If you track, you track on paper.
Simple vault containment plan
  • Create one folder: REFERENCE_INDEX
  • Create one note: MASTER_INDEX.md
  • Only index things you actually use. Ignore the rest.

Emergency folder

This is a portable folder for fast evacuation, urgent travel, or sudden bureaucracy. Keep it accessible.

Contents checklist
  • ID copies (you + spouse)
  • Insurance cards/policies summaries
  • Medication list + allergies
  • Critical contacts
  • Bank contact info (not passwords)
  • House info: shutoffs, key contacts

Troubleshooting

Problem: papers pile up
  • Install an “inbox tray” and process it weekly.
  • Reduce categories: trash/record/archive only.
  • Enforce “touch once.”
Problem: I keep checking digital anyway
  • Set a fixed weekly “library window.”
  • Move planning back to paper immediately.
  • Keep the command book physically open where you work.
Problem: I can’t find documents
  • Fix the archive index first. Index beats sorting.
  • Limit folder depth to one level.
  • Store newest at the front, always.

Core idea: paper creates selection; selection creates meaning. Digital creates accumulation; accumulation creates noise.
